# 🚀 数据库初始化执行指南

## 装修系统数据库初始化

### 📋 集合列表

初始化后将创建以下 4 个集合：

| 集合名 | 用途 | 说明 |
|--------|------|------|
| `dormitoryDecorations` | 宿舍装修配置 | 存储每个宿舍的装修布局 |
| `decorationItems` | 装饰品商城配置 | 存储所有可购买的装饰品 |
| `userDecorationBags` | 用户道具背包 | 存储用户已购买的装饰品 |
| `decorationRankingCache` | 装修排行榜缓存 | 缓存排行榜数据，提升性能 |

---

## 🔧 执行步骤

### 方法一：微信开发者工具（推荐）

1. **打开微信开发者工具**
   - 打开项目：`guagua-github`

2. **进入云开发控制台**
   - 点击顶部工具栏「云开发」按钮
   - 或菜单：工具 → 云开发

3. **进入云函数页面**
   - 左侧菜单：云函数 → 本地云函数

4. **运行初始化脚本**
   - 在云开发控制台的「数据库」页面
   - 点击右侧「数据导入」或「运行脚本」
   - 选择 `init-decoration-db.js` 文件
   - 点击「运行」

5. **等待执行完成**
   - 查看日志输出
   - 确认 4 个集合创建成功

## ⚠️ 常见问题

### Q1: 集合已存在怎么办？

**A:** 如果集合已存在，脚本会跳过创建，直接添加索引。如果想重置，可以：

1. 在云开发控制台删除对应集合
2. 重新运行初始化脚本

### Q2: 导入装饰品数据失败？

**A:** 检查以下几点：

1. 确认 `config/decoration-config.json` 文件存在
2. 确认 JSON 格式正确
3. 检查网络是否正常

### Q3: 索引创建失败？

**A:** 可能是集合中已有数据导致索引冲突。可以：

1. 清空集合数据
2. 删除集合后重新创建
