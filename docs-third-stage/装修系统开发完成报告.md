## 📊 数据库设计亮点

### 集合设计

1. **dormitoryDecorations（宿舍装修配置）**
   - 支持多个装饰品摆放
   - 记录位置、旋转角度
   - 自动计算评分和等级

2. **userDecorationItems（用户背包）**
   - 区分已购买和已放置
   - 支持多种货币类型
   - 复合索引优化查询

3. **decorationItems（装饰品配置）**
   - 只读数据，后台管理
   - 支持 4 种类型（家具/墙纸/地板/装饰）
   - 支持 4 种稀有度（普通/稀有/史诗/传说）

4. **decorationRecords（操作日志）**
   - 完整审计追溯
   - 支持回滚操作
   - 防作弊监控

### 索引优化

- **唯一索引：** dormitoryId、itemId 确保数据唯一性
- **复合索引：** userId+itemId、type+rarity 优化多条件查询
- **降序索引：** score 支持排行榜快速查询

### 评分系统

```
总分 = (基础分 + 组合加成 + 稀有度加成) × 等级系数

基础分 = Σ(每个装饰品的 score)
组合加成 = 同系列装饰品数量 × 10
稀有度加成 = rare×5 + epic×20 + legendary×50
等级系数 = 1 + (宿舍等级 - 1) × 0.1
```

## ⚠️ 风险提示

### 技术风险

1. **性能风险：** 大量装饰品同时渲染可能导致卡顿
   - 缓解：使用 Canvas 渲染、分层渲染、LOD 优化

2. **并发风险：** 多用户同时装修可能数据冲突
   - 缓解：乐观锁、事务处理、版本控制

3. **安全风险：** 客户端作弊修改评分
   - 缓解：服务器端计算、签名验证、日志监控

### 进度风险

- 测试执行需要 5-7 天
- Bug 修复可能需要额外时间
- 建议预留 2-3 天缓冲
