## 📋 一、数据库集合设计

### 1.1 dormitoryDecorations - 宿舍装修配置

**用途：** 存储每个宿舍的装修布局、装饰品摆放、评分和等级信息

**字段结构：**
```javascript
{
  _id: String,                    // MongoDB 自动生成
  dormitoryId: String,            // 宿舍 ID（唯一）
  decorations: [{                 // 已购买的装饰品列表
    itemId: String,               // 装饰品 ID
    itemType: String,             // 类型：furniture/wall/floor/decoration
    position: {                   // 位置信息
      x: Number,                  // X 坐标（网格）
      y: Number,                  // Y 坐标（网格）
      rotation: Number            // 旋转角度：0/90/180/270
    },
    purchasedAt: Number           // 购买时间戳
  }],
  layout: {                       // 宿舍布局配置
    width: Number,                // 宿舍宽度（像素）
    height: Number,               // 宿舍高度（像素）
    gridSize: Number              // 网格大小（像素）
  },
  score: Number,                  // 装修评分（0-1000）
  level: Number,                  // 宿舍等级（1-10）
  lastUpdatedAt: Number,          // 最后更新时间戳
  updatedBy: String               // 最后更新者 userId
}
```

**示例数据：**
```javascript
{
  _id: "dec_67890",
  dormitoryId: "dorm_12345",
  decorations: [
    {
      itemId: "item_001",
      itemType: "furniture",
      position: { x: 2, y: 3, rotation: 0 },
      purchasedAt: 1710662400000
    },
    {
      itemId: "item_015",
      itemType: "wall",
      position: { x: 0, y: 0, rotation: 0 },
      purchasedAt: 1710662500000
    }
  ],
  layout: {
    width: 800,
    height: 600,
    gridSize: 50
  },
  score: 750,
  level: 5,
  lastUpdatedAt: 1710748800000,
  updatedBy: "user_abc123"
}
```

**用途：** 存储所有可购买的装饰品配置信息（后台管理，用户只读）

**字段结构：**
```javascript
{
  _id: String,                    // MongoDB 自动生成
  itemId: String,                 // 装饰品 ID（唯一）
```

## 🔍 二、数据库索引设计

### 2.1 dormitoryDecorations 集合索引

| 索引名称 | 字段 | 类型 | 用途 |
|---------|------|------|------|
| idx_dormitoryId | dormitoryId | 唯一索引 | 快速查找宿舍装修 |
| idx_score_desc | score | 降序索引 | 排行榜查询 |
| idx_level | level | 普通索引 | 按等级筛选 |
| idx_lastUpdatedAt | lastUpdatedAt | 普通索引 | 同步检测 |

**创建索引命令：**
```javascript
db.dormitoryDecorations.createIndex({ dormitoryId: 1 }, { unique: true })
db.dormitoryDecorations.createIndex({ score: -1 })
db.dormitoryDecorations.createIndex({ level: 1 })
db.dormitoryDecorations.createIndex({ lastUpdatedAt: -1 })
```

---

### 2.2 userDecorationItems 集合索引

| 索引名称 | 字段 | 类型 | 用途 |
|---------|------|------|------|
| idx_user_item | userId, itemId | 复合索引 | 查询用户特定物品 |
| idx_userId | userId | 普通索引 | 查询用户背包 |

**创建索引命令：**
```javascript
db.userDecorationItems.createIndex({ userId: 1, itemId: 1 })
db.userDecorationItems.createIndex({ userId: 1 })
```

---

### 2.3 decorationItems 集合索引

| 索引名称 | 字段 | 类型 | 用途 |
|---------|------|------|------|
| idx_itemId | itemId | 唯一索引 | 快速查找物品配置 |
| idx_type_rarity | type, rarity | 复合索引 | 商城筛选 |
| idx_currency | currency | 普通索引 | 按货币类型筛选 |

**创建索引命令：**
```javascript
db.decorationItems.createIndex({ itemId: 1 }, { unique: true })
db.decorationItems.createIndex({ type: 1, rarity: 1 })
db.decorationItems.createIndex({ currency: 1 })
```

---

### 2.4 decorationRecords 集合索引

| 索引名称 | 字段 | 类型 | 用途 |
|---------|------|------|------|
| idx_dormitory_time | dormitoryId, createdAt | 复合索引 | 查询宿舍操作历史 |
| idx_user_time | userId, createdAt | 复合索引 | 查询用户操作历史 |
| idx_createdAt | createdAt | 降序索引 | 按时间排序 |

**创建索引命令：**
```javascript
db.decorationRecords.createIndex({ dormitoryId: 1, createdAt: -1 })
db.decorationRecords.createIndex({ userId: 1, createdAt: -1 })
db.decorationRecords.createIndex({ createdAt: -1 })
```

### 4.7 calculateDecorationScore - 计算装修评分

**输入：**
```javascript
{
  dormitoryId: String
}
```

**评分规则：**
```javascript
总分 = 基础分 + 组合加成 + 稀有度加成 + 等级系数

基础分 = Σ(每个装饰品的 score)
组合加成 = 同系列装饰品数量 × 10
稀有度加成 = (rare×5 + epic×20 + legendary×50)
等级系数 = 1 + (宿舍等级 - 1) × 0.1
```

**输出：**
```javascript
{
  success: Boolean,
  data: {
    baseScore: Number,
    comboBonus: Number,
    rarityBonus: Number,
    levelMultiplier: Number,
    totalScore: Number,
    newLevel: Number
  }
}
```

## ⚠️ 五、安全与风控

### 5.1 权限控制

1. **数据库权限**
   - 用户只能读写自己的背包数据
   - 宿舍装修：宿舍成员可写，其他人只读
   - 装饰品配置：仅管理员可写

2. **云函数权限**
   - 基于 OPENID 验证用户身份
   - 宿舍操作验证成员身份
   - 敏感操作记录日志

---

### 5.2 防刷机制

1. **位置验证**
   - 服务器端验证装饰品位置合法性
   - 防止穿墙、重叠等作弊行为

2. **评分计算**
   - 评分在服务器端计算
   - 客户端仅展示，不可信

3. **购买限制**
   - 单次购买数量上限
   - 每日购买次数限制（可配置）

---

### 5.3 数据一致性

1. **事务处理**
   - 积分扣减 + 物品增加：原子操作
   - 背包扣减 + 宿舍放置：原子操作
   - 失败时自动回滚

2. **并发控制**
   - 使用乐观锁（version 字段）
   - 检测并发修改

## 📈 六、性能优化

### 6.1 查询优化

1. **排行榜缓存**
   - 使用 decorationLeaderboard 集合缓存
   - 每 5 分钟更新一次
   - 支持实时计算 fallback

2. **分页查询**
   - 所有列表接口支持分页
   - 默认每页 20 条

---

### 6.2 索引优化

1. **常用查询字段建立索引**
2. **复合索引覆盖多条件查询**
3. **定期分析慢查询日志**

---

### 6.3 数据清理

1. **操作日志保留 90 天**
2. **过期数据归档**
3. **定期清理测试数据**
