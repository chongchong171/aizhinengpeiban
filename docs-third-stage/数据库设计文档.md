# 数据库设计文档 - 商业化功能

## 概述

本文档描述瓜瓜陪伴小程序商业化功能的数据库结构设计。

## 集合列表

### 1. members（成员信息）

**说明：** 存储用户基本信息和积分数据

#### 原有字段
```javascript
{
  _id: String,              // 文档 ID
  userId: String,           // 用户 OpenID
  dormitoryId: String,      // 宿舍 ID
  nickname: String,         // 昵称
  title: String,            // 称号
  customTitle: String,      // 自定义称号
  mood: String,             // 心情
  moodEmoji: String,        // 心情 Emoji
  moodTime: Number,         // 心情更新时间
  lastActiveDate: Number,   // 最后活跃时间
  joinedAt: Number          // 加入时间
}
```

#### 新增字段（商业化功能）
```javascript
{
  starlightPoints: Number,      // 当前星光值（默认：0）
  crystalDiamonds: Number,      // 当前晶钻（默认：0）
  totalStarlight: Number,       // 累计获得星光值（默认：0）
  totalCrystal: Number,         // 累计获得晶钻（默认：0）
  personalExp: Number,          // 个人经验值（默认：0）
  level: Number                 // 个人等级（默认：1）
}
```

#### 索引
- `userId`: 唯一索引
- `dormitoryId`: 普通索引

### 2. couponRedemptions（优惠券兑换记录）

**说明：** 存储用户兑换的优惠券信息

#### 字段结构
```javascript
{
  _id: String,              // 文档 ID
  userId: String,           // 用户 OpenID
  dormitoryId: String,      // 宿舍 ID
  couponType: String,       // 优惠券类型（coupon_50, coupon_60, ...）
  couponValue: Number,      // 优惠券面额（50, 60, 80, 100, 150）
  minPurchase: Number,      // 最低消费门槛（200, 400, 500, 600, 1000）
  pointsType: String,       // 使用的积分类型（starlight | crystal）
  pointsCost: Number,       // 消耗的积分数量
  status: String,           // 状态（unused | used | expired）
  expireAt: Number,         // 过期时间（时间戳）
  createdAt: Number,        // 创建时间（时间戳）
  usedAt: Number,           // 使用时间（可选）
  orderId: String           // 使用的订单 ID（可选）
}
```

#### 索引
- `userId`: 普通索引（查询用户优惠券）
- `status`: 普通索引（筛选状态）
- `expireAt`: 普通索引（检查过期）
- `userId + status + createdAt`: 复合索引（查询用户某状态优惠券）

#### 状态流转
```
unused → used（使用优惠券）
unused → expired（过期）
```

---

### 3. taskRecords（任务完成记录）

**说明：** 存储用户任务完成和奖励领取记录

#### 字段结构
```javascript
{
  _id: String,              // 文档 ID
  userId: String,           // 用户 OpenID
  dormitoryId: String,      // 宿舍 ID
  taskType: String,         // 任务类型（morning_checkin, night_checkin, ...）
  taskCategory: String,     // 任务分类（daily | weekly | dormitory）
  reward: Number,           // 奖励数量
  rewardType: String,       // 奖励类型（starlight | crystal）
  completedAt: Number       // 完成时间（时间戳）
}
```

#### 索引
- `userId`: 普通索引
- `userId + completedAt`: 复合索引（查询用户某时间段任务）
- `userId + taskType + completedAt`: 复合索引（检查任务是否完成）

## 数据库操作

### 初始化脚本

位置：`cloudfunctions/initDatabase/index.js`

**功能：**
1. 创建 couponRedemptions 集合
2. 创建 taskRecords 集合
3. 为所有集合添加索引
4. 更新现有 members 记录的积分字段

**执行方式：**
```javascript
// 在云开发控制台或云函数中执行
const init = require('./cloudfunctions/initDatabase');
await init.initDatabase();
await init.updateMembersFields();
```

---

### 常用查询

#### 获取用户积分信息
```javascript
const user = await db.collection('members')
  .where({ userId: OPENID })
  .get();
```

#### 获取用户优惠券列表
```javascript
const coupons = await db.collection('couponRedemptions')
  .where({
    userId: OPENID,
    status: 'unused'
  })
  .orderBy('createdAt', 'desc')
  .get();
```

#### 检查任务是否已完成
```javascript
const today = new Date();
today.setHours(0, 0, 0, 0);
const todayStart = today.getTime();
const todayEnd = todayStart + 24 * 60 * 60 * 1000;

const record = await db.collection('taskRecords')
  .where({
    userId: OPENID,
    taskType: 'morning_checkin',
    completedAt: _.gte(todayStart).and(_.lt(todayEnd))
  })
  .get();
```

#### 统计本月已兑换优惠券数量
```javascript
const startOfMonth = new Date();
startOfMonth.setDate(1);
startOfMonth.setHours(0, 0, 0, 0);

const count = await db.collection('couponRedemptions')
  .where({
    userId: OPENID,
    createdAt: _.gte(startOfMonth.getTime()),
    status: _.in(['unused', 'used'])
  })
  .count();
```

## 数据迁移

### 从旧版本升级

如果已有 members 数据，需要执行以下迁移：

```javascript
// 更新所有现有成员记录
const members = await db.collection('members').get();

for (const member of members.data) {
  if (!member.starlightPoints) {
    await db.collection('members').doc(member._id).update({
      data: {
        starlightPoints: 0,
        crystalDiamonds: 0,
        totalStarlight: 0,
        totalCrystal: 0,
        personalExp: 0,
        level: 1
      }
    });
  }
}
```

## 性能优化

### 索引策略
- 所有查询字段都建立索引
- 复合索引用于多条件查询
- 定期检查和优化索引

### 查询优化
- 使用字段投影减少数据传输
- 分页查询大数据集
- 避免全表扫描

### 数据清理
- 定期清理过期优惠券（可选）
- 归档历史任务记录（可选）
