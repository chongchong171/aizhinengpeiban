## 📊 第一部分：双轨积分制设计

### 1.1 积分类型

| 积分类型 | 名称 | 图标 | 获取方式 | 消耗方式 | 有效期 |
|---------|------|------|---------|---------|--------|
| **活跃积分** | 星光值 | 🌟 | 每日任务、每周挑战、宿舍集体任务 | 兑换优惠券、虚拟道具 | 永久有效 |
| **消费积分** | 晶钻 | 💎 | 充值、付费购买 | 兑换专属定制、限定道具 | 永久有效 |

**设计思路：**
- 星光值（免费）→ 保证免费用户体验，提升活跃度
- 晶钻（付费）→ 商业化变现路径
- 双轨独立 → 避免积分通胀

### 2.1 members 集合新增字段

在原有基础上增加以下字段：

```javascript
{
  // 原有字段
  userId: String,
  dormitoryId: String,

  // 新增积分字段

  totalStarlight: Number,       // 累计星光值（默认 0）
  totalCrystal: Number,         // 累计晶钻（默认 0）
  
  // 原有经验值字段
  personalExp: Number,          // 个人经验值
  totalExp: Number,             // 累计经验值
  level: Number,                // 个人等级
  
  // 活跃字段

  lastCheckinDate: Number,      // 最后签到日期戳
  
  // 任务进度
  dailyTasks: Object,           // 每日任务进度
  weeklyTasks: Object,          // 每周任务进度
  achievements: Array           // 已解锁成就列表
}
```

### 2.2 couponRedemptions 集合（新增）

**用途：** 记录用户积分兑换优惠券的流水

**字段结构：**
```javascript
{
  _id: String,                  // 记录 ID
  userId: String,               // 用户 openid
  dormitoryId: String,          // 宿舍 ID
  couponType: String,           // 优惠券类型（如：coupon_50_200）
  couponValue: Number,          // 优惠券面额（元）
  minPurchase: Number,          // 使用门槛（元）
  pointsType: String,           // 积分类型：starlight | crystal
  pointsCost: Number,           // 花费积分数量
  status: String,               // 状态：unused | used | expired
  expireAt: Number,             // 过期时间戳
  usedAt: Number,               // 使用时间戳（可选）
  createdAt: Number             // 创建时间戳
}
```

**索引建议：**
- userId + status（复合索引）
- userId + expireAt（复合索引）
- createdAt（降序）

### 2.3 taskRecords 集合（新增）

**用途：** 记录用户任务完成流水

**字段结构：**
```javascript
{
  _id: String,
  userId: String,
  taskId: String,               // 任务 ID
  taskType: String,             // daily | weekly | achievement
  rewardType: String,           // starlight | crystal | exp
  rewardAmount: Number,         // 奖励数量
  completedAt: Number,          // 完成时间戳
  claimed: Boolean,             // 是否已领取奖励
  claimedAt: Number             // 领取时间戳
}
```

## 🔧 第三部分：云函数开发

### 3.1 getPointsShop - 获取积分商城

**输入：** 无  
**输出：**
```javascript
{
  success: Boolean,
  data: {
    userPoints: {
      starlight: Number,
      crystal: Number
    },
    coupons: [
      {
        couponType: String,
        couponValue: Number,
        minPurchase: Number,
        pointsCost: Number,
        pointsType: String,
        discount: String,
        description: String,
        canRedeem: Boolean      // 是否可兑换（积分足够 + 未达上限）
      }
    ],
    monthlyLimit: Number,       // 每月兑换限制（2 张）
    redeemedThisMonth: Number,  // 本月已兑换数量
    canRedeemMore: Boolean      // 是否还能兑换
  }
}
```

### 3.2 redeemCoupon - 兑换优惠券

**输入：**
```javascript
{
  couponType: String,           // 优惠券类型
  dormitoryId: String,          // 宿舍 ID
  pointsType: String            // starlight | crystal
}
```

**处理流程：**
1. 验证参数完整性
2. 获取用户信息（星光值/晶钻）
3. 检查本月已兑换数量（防刷）
4. 检查积分是否足够
5. 开启事务
6. 扣减用户积分
7. 创建优惠券记录（30 天有效期）
8. 记录积分流水
9. 提交事务
10. 返回结果

**输出：**
```javascript
{
  success: Boolean,
  message: String,
  data: {
    couponId: String,
    couponValue: Number,
    pointsCost: Number,
    expireAt: Number
  }
}
```

### 3.3 getUserCoupons - 获取用户优惠券

**输入：**
```javascript
{
  status: String                // 'unused' | 'used' | 'expired' | undefined（全部）
}
```

**输出：**
```javascript
{
  success: Boolean,
  data: {
    all: [...],
    unused: [...],
    used: [...],
    expired: [...]
  }
}
```

### 3.4 useCoupon - 使用优惠券

**输入：**
```javascript
{
  couponId: String
}
```

**处理流程：**
1. 验证优惠券存在
2. 检查优惠券归属
3. 检查优惠券状态
4. 检查是否过期
5. 更新状态为 used

**输出：**
```javascript

  message: String,
```

**输入：**

## ⚠️ 第六部分：安全与风控

### 6.1 防刷机制

1. **每人每月最多兑换 2 张优惠券**
   - 基于自然月计算
   - 查询 couponRedemptions 集合统计

2. **任务防刷**
   - 服务器端验证任务完成条件
   - 设置每日获取上限
   - 异常行为检测

3. **积分变动事务处理**
   - 积分扣减和优惠券创建必须原子操作
   - 失败时自动回滚

### 6.2 权限控制

1. **数据库权限**
   - 用户只能读写自己的数据
   - 仅管理员可查看兑换记录

2. **云函数权限**
   - 基于 OPENID 验证用户身份
   - 敏感操作需要二次验证

### 6.3 错误处理

1. **友好提示**
   - 积分不足：星光值不足哦~
   - 已达上限：本月已达兑换上限
   - 优惠券过期：优惠券已过期

2. **日志记录**
   - 所有积分变动记录日志
   - 异常操作告警
