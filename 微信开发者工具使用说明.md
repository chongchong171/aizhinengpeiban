# 微信开发者工具使用说明

## 📱 如何查看最新的修改

### 问题原因
微信开发者工具可能打开的是**源目录**，而不是**编译后的 dist 目录**。

### 正确操作步骤

#### 方法 1：重新导入项目（推荐）

1. **关闭当前项目**
   - 在微信开发者工具中，点击左上角的项目名称
   - 选择"关闭当前项目"

2. **导入新项目**
   - 点击"+"号或"导入项目"
   - **项目路径**：`D:\aizhinengpeiban\dist`
   - **AppID**：使用测试号或您自己的 AppID
   - 点击"导入"

3. **查看效果**
   - 导入后会自动编译
   - 在模拟器中可以看到最新的修改

#### 方法 2：切换到 dist 目录

如果您已经打开了项目，可以尝试：

1. 点击微信开发者工具左上角的"项目"菜单
2. 选择"打开其他项目"
3. 选择 `D:\aizhinengpeiban\dist` 目录

### 注意事项

⚠️ **重要**：
- 每次修改代码后，都需要重新编译
- 运行编译脚本：`powershell -ExecutionPolicy Bypass -File .\build.ps1`
- 然后在微信开发者工具中刷新（Ctrl+R）

### 快速编译命令

在 Trae 终端中运行：
```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1
```

### 如果还是看不到变化

1. **清除缓存**
   - 微信开发者工具 → 工具 → 清除缓存 → 清除全部缓存

2. **重新编译**
   ```powershell
   # 删除旧的 dist
   Remove-Item -Path "D:\aizhinengpeiban\dist" -Recurse -Force
   
   # 重新编译
   powershell -ExecutionPolicy Bypass -File .\build.ps1
   ```

3. **重新导入项目**
   - 关闭当前项目
   - 重新导入 `D:\aizhinengpeiban\dist`

## 常见问题

**Q: 为什么要用 dist 目录？**
A: 因为源目录包含了一些开发文件和文档，dist 目录只包含小程序运行必需的文件，更干净、更高效。

**Q: 每次修改都要重新编译吗？**
A: 是的，修改完代码后需要运行编译脚本，然后在微信开发者工具中刷新。

**Q: 可以直接打开源目录吗？**
A: 理论上可以，但不推荐。源目录包含一些不必要的文件，可能导致编译错误或性能问题。
