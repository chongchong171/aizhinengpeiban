## 1. 系统概述

## 2. 数据库设计

### 2.1 dormitoryDecorations 集合

**用途：** 存储宿舍装修配置

```javascript
{
  _id: String,                    // 记录 ID（自动生成）
  dormitoryId: String,            // 宿舍 ID（唯一索引）
  decorations: [{                 // 已购买的装饰品列表
    itemId: String,               // 道具 ID
    itemType: String,             // 类型：furniture/wall/floor/decoration
    position: {                   // 位置信息
      x: Number,                  // X 坐标（像素）
      y: Number,                  // Y 坐标（像素）
      rotation: Number            // 旋转角度（0/90/180/270）
    },
    purchasedAt: Number           // 购买时间戳
  }],
  layout: {                       // 宿舍布局配置
    width: Number,                // 画布宽度（默认 800）
    height: Number,               // 画布高度（默认 600）
    gridSize: Number              // 网格大小（默认 50）
  },
  score: Number,                  // 装修评分（0-1000）
  lastUpdatedAt: Number,          // 最后更新时间戳
  updatedBy: String,              // 最后更新的用户 openid
  createdAt: Number               // 创建时间戳
}
```

**索引设计：**
- `dormitoryId`（唯一索引）- 快速查询宿舍装修
- `score`（降序）- 排行榜查询
- `lastUpdatedAt`（降序）- 最新装修查询

---

### 2.2 decorationItems 集合

**用途：** 存储装饰品商城配置

```javascript
{
  _id: String,                    // 记录 ID
  itemId: String,                 // 道具 ID（唯一索引）
  name: String,                   // 道具名称
  description: String,            // 道具描述
  type: String,                   // 类型：furniture/wall/floor/decoration
  subtype: String,                // 具体类型：bed/desk/chair/etc
  price: Number,                  // 价格
  currency: String,               // 货币类型：starlight/crystal
  score: Number,                  // 装饰评分（用于计算总分）
  image: String,                  // 图标 URL
  rarity: String,                 // 稀有度：common/rare/epic/legendary
  isLimited: Boolean,             // 是否限定（默认 false）
  limitEndTime: Number,           // 限定结束时间戳（可选）
  sort: Number,                   // 排序权重（默认 0）
  isEnabled: Boolean              // 是否启用（默认 true）
}
```

**索引设计：**
- `itemId`（唯一索引）- 道具查询
- `type_rarity`（复合索引）- 类型 + 稀有度筛选
- `price`（升序）- 价格排序

---

### 2.3 userDecorationBags 集合

**用途：** 存储用户已购买的装饰品（背包）

```javascript
{
  _id: String,                    // 记录 ID
  userId: String,                 // 用户 openid
  itemId: String,                 // 道具 ID
  quantity: Number,               // 数量（默认 1）
  purchasedAt: Number,            // 购买时间戳
  usedAt: Number,                 // 使用时间戳（可选）
  isInUse: Boolean                // 是否正在使用（默认 false）
}
```

**索引设计：**
- `userId_itemId`（唯一复合索引）- 用户道具查询
- `userId`（索引）- 用户背包查询

---

### 2.4 decorationRankingCache 集合

**用途：** 缓存装修排行榜数据（性能优化）

```javascript
{
  _id: String,                    // 记录 ID
  cacheKey: String,               // 缓存键：ranking:daily/weekly/monthly/all
  rankingData: Array,             // 排行榜数据
  generatedAt: Number,            // 生成时间戳
  expireAt: Number                // 过期时间戳

```

**索引设计：**
- `cacheKey`（唯一索引）- 缓存查询
- `expireAt`（索引）- 过期清理

### 3.2 核心流程

#### 3.2.1 购买装饰品流程

```
用户点击购买
    ↓
buyDecorationItem 云函数
    ↓
1. 验证参数（itemId, currency, quantity）
    ↓
2. 查询道具信息（价格、库存）
    ↓
3. 查询用户积分余额
    ↓
4. 验证积分是否足够
    ↓
5. 开启事务（原子操作）
    ↓
5.1 扣减用户积分
5.2 更新用户背包（新增 or 累加）
5.3 记录积分流水
    ↓
6. 提交事务
    ↓
7. 返回购买结果
```

#### 3.2.2 保存装修流程

```
用户编辑完成，点击保存
    ↓
saveDormitoryDecoration 云函数
    ↓
1. 验证参数（dormitoryId, decorations）
    ↓
2. 验证用户宿舍归属
    ↓
3. 验证装饰品所有权（防刷）
    ↓
4. 计算装修评分
    ↓
5. 保存/更新装修配置
    ↓
6. 返回保存结果 + 评分
```

## 4. 装修评分算法

### 4.1 评分维度

根据官方文档要求，评分由以下 4 个维度组成：

| 维度 | 权重 | 满分 | 说明 |
|------|------|------|------|
| 装饰品总价值 | 30% | 300 | 所有装饰品价格总和 |
| 装饰品稀有度 | 30% | 300 | 稀有度加权平均 |
| 装饰品数量 | 20% | 200 | 装饰品总数量 |
| 搭配协调性 | 20% | 200 | 类型分布均衡度 |

### 4.2 评分公式

```javascript
总分 = 价值分数 × 0.3 + 稀有度分数 × 0.3 + 数量分数 × 0.2 + 协调性分数 × 0.2
```

### 4.3 详细计算

#### 4.3.1 价值分数（300 分满分）

```javascript
价值分数 = min(300, (总价值 / 10000) × 300)
// 假设 10000 星光值/晶钻为满分
```

#### 4.3.2 稀有度分数（300 分满分）

```javascript
稀有度权重 = {
  common: 1,
  rare: 3,
  epic: 6,
  legendary: 10
}

平均稀有度 = Σ(各道具稀有度权重) / 道具数量
稀有度分数 = min(300, (平均稀有度 / 5) × 300)
// 假设平均稀有度 5 为满分
```

#### 4.3.3 数量分数（200 分满分）

```javascript
数量分数 = min(200, (道具数量 / 20) × 200)
// 假设 20 个装饰品为满分
```

#### 4.3.4 协调性分数（200 分满分）

```javascript
// 计算各类型道具数量的方差
类型分布 = {
  furniture: 数量，
  wall: 数量，
  floor: 数量，
  decoration: 数量
}

平均值 = 总数量 / 类型数
方差 = Σ(各类型数量 - 平均值)² / 类型数

协调性分数 = max(0, 200 - (方差 × 10))
// 方差越小，分数越高
```

### 4.4 评分等级

| 总分范围 | 等级 | 称号 |
|---------|------|------|
| 800-1000 | legendary | 传说设计师 |
| 600-799 | epic | 史诗设计师 |
| 400-599 | rare | 精英设计师 |
| 200-399 | uncommon | 优秀设计师 |
| 0-199 | common | 新手设计师 |

## 5. 安全与风控

### 5.1 防刷机制

#### 5.1.1 装饰品所有权验证

在保存装修时，验证用户是否拥有该装饰品：

```javascript
// 查询用户背包
const userBag = await db.collection('userDecorationBags')
  .where({ userId: OPENID, quantity: _.gt(0) })
  .get();

// 验证每个装饰品
for (const dec of decorations) {
  if (!ownedItemIds.has(dec.itemId)) {
    throw new Error(`未拥有装饰品：${dec.itemId}`);
  }
}
```

#### 5.1.2 宿舍归属验证

只有同宿舍成员才能修改装修：

```javascript
const user = await db.collection('members')
  .where({ userId: OPENID })
  .get();

if (user.dormitoryId !== dormitoryId) {
  throw new Error('不属于该宿舍');
}
```

### 5.2 事务处理

购买装饰品使用事务保证原子性：

```javascript
const transaction = await db.startTransaction();

try {
  // 扣减积分
  await transaction.collection('members').doc(userDocId).update({...});
  
  // 更新背包
  await transaction.collection('userDecorationBags').add({...});
  
  // 记录流水
  await transaction.collection('pointsTransactions').add({...});
  
  await transaction.commit();
} catch (e) {
  await transaction.rollback();
  throw e;
}
```

### 5.3 权限控制

- 用户只能查看自己的背包
- 同宿舍成员可以查看宿舍装修
- 只有宿舍成员可以修改装修
- 排行榜公开可见

## 6. 性能优化

### 6.1 排行榜缓存策略

#### 6.1.1 缓存配置

| 排行榜类型 | 缓存时间 | 说明 |
|-----------|---------|------|
| 日榜 | 5 分钟 | 变化频繁 |
| 周榜 | 10 分钟 | 中等频率 |
| 月榜 | 15 分钟 | 较低频率 |
| 总榜 | 30 分钟 | 变化最慢 |

#### 6.1.2 缓存逻辑

```javascript
// 1. 尝试从缓存获取
const cachedData = await getFromCache(cacheKey);
if (cachedData) {
  return paginatedData(cachedData, limit, offset);
}

// 2. 缓存未命中，查询数据库
const rankingData = await queryFromDatabase();

// 3. 写入缓存
await cacheRanking(cacheKey, rankingData, expireSeconds);
```

### 6.2 数据库索引优化

- 所有查询字段都建立索引
- 复合索引优化多条件查询
- 降序索引优化排序查询

### 6.3 分页查询

排行榜使用分页查询，避免一次性加载大量数据：

```javascript
const result = await db.collection('dormitoryDecorations')
  .where(query)
  .orderBy('score', 'desc')
  .skip(offset)
  .limit(limit)
  .get();
```
